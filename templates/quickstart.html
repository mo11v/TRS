📡 Scenario 2 — ESP32 Connected
1
Flash ESP32 v11
  1. افتح smart_torque_monitor_esp32_v11.ino في Arduino IDE
  2. Library Manager → ابحث عن WebSockets by Markus Sattler → Install
  3. Board: ESP32 Dev Module
  4. Upload → انتظر LCD يعرض:
    STA: 192.168.10.70 AP: 192.168.99.1
2
وصّل TRS بالـ ESP32
الـ TRS والـ ESP32 لازم يكونوا على نفس الـ LAN (MTT WiFi):
ESP32 STA IP: 192.168.10.70 (MTT network) TRS Server: نفس الشبكة TCP Port: 8888 (Python v6 sender) Test الاتصال: GET http://192.168.10.70/data → JSON WS ws://192.168.10.70:81 → real-time
شغّل Python v6 على نفس الـ MTT network — هيتصل بـ ESP32 تلقائي على port 8888
3

  3. الـ gauges هتبدأ تتحرك real-time
  4. الـ WebSocket dot لازم يبقى ● أخضر
🚀 Scenario 3 — Full Production Stack
0
Architecture
MTT Device ↓ UDP WiFi ESP32 v11 ←── STA: متصل بشبكة MTT │ └── AP: TRS-FIELD-001 (للموبايل) ↓ TCP:8888 Python v6 (على PC متصل بـ MTT) ↓ processes data → sends to ESP32 Moبايل ←── متصل بـ TRS-FIELD-001 ↓ WebSocket/HTTP ← يسحب من ESP32 ↓ 4G Render (TRS Server) ├── Firestore (persistent DB) └── Cloudinary (PDF storage) ↓ WebSocket Dashboard (أي browser في العالم)
1
GitHub + Render Deploy
git init && git add . && git commit -m "TRS v1" git remote add origin https://github.com/YOUR/trs-platform.git git push -u origin main # ثم: render.com → New Web Service → Connect repo
الـ render.yaml جوا المشروع — Render بيقرأه تلقائي
2
Firebase + Cloudinary
  1. console.firebase.google.com → New Project → Firestore Database
  2. Project Settings → Service Accounts → Generate Key → انسخ JSON
  3. cloudinary.com → Sign up → انسخ Cloud Name + API Key + Secret
  4. Render Dashboard → Environment → ضيف الـ variables
التفاصيل كاملة في SETUP.md جوا الـ ZIP
3
Mobile Bridge
  1. افتح trs_field_bridge_v2.html في Chrome على الموبايل
  2. WiFi → اتصل بـ TRS-FIELD-001 (pass: trs12345)
  3. Connect tab → ادخل TRS URL → اضغط ▶ Start
  4. Cloud tab → ادخل Firebase credentials → Test
  5. الـ 3 dots المفروض يبقوا خضر 🟢🟢🟢
